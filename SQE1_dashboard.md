## Error code definitions

- R: rule not known.
- F: fact misread.
- E: exception missed.
- P: procedure sequence error.
- ETH: ethics issue missed.
- CALC: calculation/date error.

## Pass strategy

1. Build black-letter rule recall first.
2. Convert rules into client-fact application.
3. Drill single-best-answer elimination.
4. Keep an error log that directly updates rule sheets.
5. Treat ethics as pervasive, not as a separate final topic only.
